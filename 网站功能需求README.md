# 南京灵动信息技术有限公司官网功能需求 README

## 项目概述

- **网站名称：** 南京灵动信息技术有限公司官方网站
- **网站域名：** `www.ilingdong.cn`
- **建设范围：** 前端开发、后端开发、数据库搭建、接口对接、数据迁移、部署上线

## 技术要求

- **前端技术栈：** Vue
- **后端技术栈：** Go
- **数据库：** MySQL
- **临时测试数据库：**
  - 地址：`180.102.24.183`
  - 用户名：`root`
  - 密码：`jny@2216446`
- **文件存储：** 图片和上传文件使用服务器本地存储，默认目录为当前运行程序目录下的 `/oss`，不使用任何对象存储服务
- **访问协议：** 全站支持 HTTPS
- **浏览器兼容：** Chrome、Edge、Safari、Firefox 最新版本
- **响应式适配：**
  - PC：1920、1440、1024
  - 平板：768
  - 手机：375、414

## 前端功能需求

### 页面渲染

- 根据 UI 设计方案完成全部页面前端开发和渲染。
- 页面视觉效果需符合确认后的设计稿。
- 支持适度交互动效，包括 hover 效果、滚动加载等。

### 导航与布局

- 顶部导航固定展示，包含 Logo、5 项一级菜单和右侧咨询按钮。
- 移动端顶部导航收起为汉堡菜单。
- 底部导航展示公司信息、栏目链接、联系方式、版权信息和备案号。

### 表单提交

首页底部提供简版表单：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 姓名 | 是 | 访客姓名 |
| 电话 | 是 | 联系电话 |
| 公司 | 是 | 公司名称 |

合作页提供完整版表单：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 姓名 | 是 | 访客姓名 |
| 公司 | 是 | 公司名称 |
| 职位 | 否 | 访客职位 |
| 电话 | 是 | 联系电话 |
| 邮箱 | 否 | 联系邮箱 |
| 需求描述 | 否 | 合作需求说明 |
| 资料勾选 | 否 | 用户感兴趣的资料或服务 |

表单提交后应完成以下动作：

- 将数据写入后台数据库。
- 自动触发邮件通知。
- 在后台记录来源页面和提交时间。

### 列表筛选与分页

- 服务案例列表页支持按行业、平台、打法筛选。
- 筛选维度命名需在开发前确认，候选名称为「打法」或「赛道」。
- 筛选过程应支持前端无刷新加载。
- 营销资讯列表页支持按灵动动态、营销洞察、行业资讯筛选。
- 服务案例列表页和营销资讯列表页支持分页，每页展示 9 至 12 条。

### FAQ 交互

- 合作页 FAQ 支持折叠和展开。
- 默认状态、展开状态和多条同时展开规则需在 UI 设计稿或开发确认阶段明确。

## 后端与 CMS 需求

后台管理系统需支持网站内容的增删改查，并覆盖以下模块。

### 案例管理

| 字段 | 说明 |
| --- | --- |
| 标题 | 案例名称 |
| 行业标签 | 用于行业筛选 |
| 封面图 | 列表页展示图 |
| 详情内容 | 案例详情正文 |
| 核心数据 | 案例数据亮点 |

### 资讯管理

| 字段 | 说明 |
| --- | --- |
| 标题 | 资讯标题 |
| 分类 | 灵动动态、营销洞察、行业资讯 |
| 封面图 | 列表页展示图 |
| 正文 | 资讯详情内容 |
| 发布时间 | 前台展示时间和排序依据 |

### Banner 管理

- 支持首页 Banner 图片替换。
- 支持首页 Banner 文案编辑。

### 静态页面内容编辑

- 支持服务页、关于我们页等静态页面内容编辑。
- 编辑能力应提供富文本编辑器。

### 表单数据管理

- 后台可查看所有表单提交记录。
- 表单记录至少包含姓名、电话、公司、需求、提交时间、来源页面。
- 支持将表单数据导出为 Excel。
- 支持标记「已处理」和「未处理」。

### 邮件通知

- 表单提交后自动发送通知邮件至指定邮箱。
- 邮件接收人支持配置多个。

### 权限管理

后台至少支持以下角色：

| 角色 | 权限说明 |
| --- | --- |
| 超级管理员 | 拥有全部管理权限 |
| 编辑 | 负责内容录入和编辑 |
| 运营 | 负责表单查看、内容运营等操作 |

不同角色应具备不同操作权限。

### 操作日志

后台需记录关键操作日志，包括：

- 登录
- 新增
- 编辑
- 删除

### 基础设置

后台支持配置以下站点基础信息：

- 网站标题
- Logo
- 联系方式
- 底部版权

### SEO 设置

- 支持各页面独立设置 TDK。
- TDK 包含标题（Title）、描述（Description）和关键词（Keywords）。

## 接口与数据需求

### 表单提交接口

- 前端表单数据提交至后端。
- 后端保存表单数据。
- 后端触发邮件通知。
- 接口需具备基础安全校验和重复提交防护。

### 文件上传与图片存储

- 后台上传的 Banner 图、案例封面图、资讯封面图、富文本图片等文件统一存储在服务器本地目录。
- 默认存储目录为当前运行程序目录下的 `/oss`。
- 不使用任何对象存储服务，包括但不限于 OSS、S3、COS、OBS 等。
- 后端需提供本地文件上传接口，并返回可访问的文件 URL。
- 上传文件需按业务类型或日期分目录存储，避免单目录文件过多。
- 上传接口需限制文件类型、文件大小和文件名安全性。
- 删除内容时，应根据业务规则决定是否同步清理无引用图片。

### 数据统计接入

- 预留第三方数据统计工具接入能力。
- 前端需支持嵌入统计代码。

### 数据迁移

如存在旧网站数据，需完成：

- 数据比对
- 数据移植
- 数据检查
- 迁移后测试

## 非功能需求

### 性能要求

- 首页首次加载时间在标准网络环境下不超过 3 秒。
- 后台管理系统操作响应时间不超过 2 秒。

### 安全要求

- 防护 SQL 注入。
- 防护 XSS 跨站脚本攻击。
- 防护 CSRF 跨站请求伪造。
- 全站启用 HTTPS。
- 安全漏洞扫描不得存在高危风险项。

### 可用性

- 网站支持 7×24 小时访问，服务器故障等不可抗因素除外。

## 验收标准

网站验收需满足以下条件：

- 所有页面在 Chrome、Edge、Safari、Firefox 最新版本下正常显示。
- 移动端在 iOS 和 Android 主流机型下适配正常。
- 页面加载速度在标准网络环境下不超过 3 秒。
- 后台管理系统所有功能正常运行。
- 不存在程序错误和链接错误。
- 安全漏洞扫描无高危风险项。
- 视觉效果符合确认后的 UI 设计稿。

## 交付物清单

- 网站完整源代码，包括前端和后端完整代码。
- 数据库文件，包括表结构和初始数据。
- 部署文档，包括环境配置说明和部署步骤。
- 后台操作说明，可采用图文或视频形式。
- 测试报告。

## 部署与上线

- 在指定服务器上部署网站。
- 完成服务器环境配置。
- 完成域名解析配置。
- 安装 SSL 证书。
- 如存在旧站数据，完成数据迁移。
- 上传至网络服务器测试，并保证网站正常运行和访问。

## 待确认事项

以下内容建议开发前确认：

- 邮件通知接收邮箱列表。
- 首页与合作页表单提交后的前台提示文案。
- 案例筛选维度使用「打法」还是「赛道」。
- CMS 各角色的详细权限边界。
- 数据统计工具类型，例如百度统计、Google Analytics 或其他工具。
- 是否存在旧网站数据迁移，以及迁移字段范围。
- 备案号、版权信息、联系方式等底部展示内容。
